[name]

An exporter for glTF 2.0 assets.

Extensions

GLTFExporter supports the following glTF 2.0 extension:

  • KHR_materials_unlit

Code Example

const exporter = new v3d.GLTFExporter(); exporter.parse( app.scene, function(gltf) { console.log(gltf); }, function(error) { console.log('glTF export failed!'); }, options );

[method:undefined parse]([param:Object3D input], [param:Function onCompleted], [param:Function onError], [param:Object options])

[page:Object input]
Scene or objects to export.
[page:Function onCompleted]
Will be called when the export completes. The argument will be the generated glTF JSON or binary [page:ArrayBuffer].
[page:Function onError]
Will be called if there are any errors during the glTF generation.
[page:Options options]
Export options:
  • [page:Boolean trs] — export position, rotation, and scale instead of matrix per node. Default is `false`.
  • [page:Boolean onlyVisible] — export only visible objects. Default is `true`.
  • [page:Boolean binary] — export in binary (.glb) format, returning an [page:ArrayBuffer]. Default is `false`.
  • [page:Integer maxTextureSize] — restricts the image maximum size (both width and height) to the given value. Default is `Infinity`.
  • [page:Array animations] — array of [page:AnimationClip AnimationClip]. List of animations to be included in the export.
  • [page:Boolean includeCustomExtensions] — export custom glTF extensions defined on an object's `userData.gltfExtensions` property. Default is `false`.

Export scene/objects to to glTF 2.0 format.
