[name]

Этот класс представляет собой абстракцию WebXR Device API и внутренне используется [page:WebGLRenderer]. [name] также предоставляет публичный интерфейс, который позволяет пользователям включать/выключать XR и выполнять связанные с XR задачи, такие как например, извлечение контроллеров.

Свойства

[property:Boolean enabled]

Это свойство уведомляет рендерер о готовности к XR-рендерингу. По умолчанию *false*. Установите его в *true*, если вы собираетесь использовать XR в своем приложении.

[property:Boolean isPresenting]

Является ли презентация XR активной или нет. По умолчанию *false*. Это свойство доступно только для чтения и автоматически устанавливается [name].

Методы

[method:Group getController]([param:Integer index])

[page:Integer index] — Индекс контроллера.

Возвращает [page:Group], представляющий так называемое пространство *целевых лучей* контроллера. Используйте это пространство для визуализации 3D-объектов, которые помогают пользователю в задачах наведения, таких как взаимодействие с пользовательским интерфейсом.

[method:Group getControllerGrip]([param:Integer index])

[page:Integer index] — Индекс контроллера.

Возвращает [page:Group], представляющий так называемое пространство *grip* контроллера. Используйте это пространство, если пользователь собирается держать другие 3D объекты, например световой меч.

Примечание: Если вы хотите одновременно показать что-то в руке пользователя и предложить луч наведения, вам нужно прикрепить удерживаемый объект к группе, возвращаемой [page:.getControllerGrip](), а луч - к группе, возвращаемой [page:.getController](). Идея заключается в том, чтобы иметь две разные группы в двух разных координатных пространствах для одного и того же контроллера WebXR.

[method:void setFramebufferScaleFactor]([param:Float framebufferScaleFactor])

[page:Float framebufferScaleFactor] — Коэффициент масштабирования буфера фрейма для установки.

Указывает коэффициент масштабирования, используемый при определении размера буфера фрейма при рендеринге на XR-устройстве. Значение задается относительно разрешения дисплея устройства XR по умолчанию. По умолчанию *1*. Значение *0.5* определяет фреймбуфер с 50% от исходного разрешения дисплея.

Примечание: Невозможно изменить коэффициент масштабирования буфера фрейма при отображении XR-контента.

[method:void setReferenceSpaceType]([param:String referenceSpaceType])

[page:String referenceSpaceType] — Тип пространства референса для установки.

Может использоваться для настройки пространственных отношений с физическим окружением пользователя. В зависимости от того, как пользователь перемещается в трехмерном пространстве, установка соответствующего опорного пространства может улучшить отслеживание. По умолчанию *local-floor*. Пожалуйста, посмотрите в [link:https://developer.mozilla.org/en-US/docs/Web/API/XRReferenceSpaceType MDN] возможные значения и случаи их использования.

Примечание: Невозможно изменить тип опорного пространства во время представления XR-контента.
